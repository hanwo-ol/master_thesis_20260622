딥러닝 모델, 특히 생성 모델(GAN)에서 정규화(Normalization)의 주된 목적은 학습의 안정화를 꾀하고, 각 층의 입력 데이터 분포가 학습 과정에서 계속 변화하는 현상(Internal Covariate Shift)을 완화하는 것입니다.

이 과정에서 일반적으로 사용되는 기법들과 연구자님께서 사용하신 SPADE는 정규화 모수($\gamma, \beta$)를 산출하고 적용하는 공간적 범위(Spatial Domain)에서 결정적인 차이를 보입니다. 통계학적 관점에서 이를 명확히 구분하여 설명해 드립니다.

---

### **1. 전역적/채널별 정규화 기법 (Global/Channel-wise Normalization)**

가장 보편적인 기법들은 이미지 전체(공간적 차원 $H, W$)에 걸쳐 하나의 평균($\mu$)과 분산($\sigma$)을 산출한 후, 이를 채널별로 동일하게 적용하는 전역적 변환(Affine Transformation)을 수행합니다.

$$\text{Norm}(x) = (1+\gamma) \odot \frac{x - \mu(x)}{\sigma(x)} + \beta$$

* **배치 정규화 (Batch Normalization, BN):** 미니 배치 차원을 포함하여 통계량을 산출합니다. 학습 단계의 배치 크기에 의존적입니다.
* **인스턴스 정규화 (Instance Normalization, IN):** 각 개별 이미지(샘플) 내에서 채널별로 통계량을 산출합니다. 화풍 변환(Stylization)과 같은 생성 과제에서 BN보다 우수한 성능을 보이며 표준적으로 쓰입니다.
* **레이어 정규화 (Layer Normalization, LN):** 각 샘플 내에서 모든 채널에 걸쳐 하나의 통계량을 산출합니다.

이들은 입력 이미지 전체를 하나의 분포로 간주하고, 이를 표준화(Standardization)한 뒤 채널별 고정된 스칼라 값($\gamma, \beta$)으로 일률적인 아핀 변환을 수행합니다. 따라서 이미지 내의 공간적인 구조 정보는 변환 과정에서 크게 고려되지 않습니다.

---

### **2. 공간 적응형 정규화 (Spatially-Adaptive Normalization, SPADE)**

이에 반해, SPADE는 입력되는 조건부 문맥 맵(Context Map, $c$)을 기반으로 정규화 모수 $\gamma$와 $\beta$를 스칼라가 아닌 **텐서(Tensor)** 형태로 *학습*하여 적용합니다.

$$\text{SPADENorm}(h, c) = (1+\gamma_{\boldsymbol{s}}(c)) \odot \text{IN}(h) + \beta_{\boldsymbol{s}}(c)$$


(여기서 $\text{IN}$은 Instance Normalization, $\boldsymbol{s}$는 공간 위치(픽셀)를 의미하는 첨자)

 SPADE의 핵심 혁신은 정규화 모수 $\gamma_s(c)$와 $\beta_s(c)$가 단순한 가중치가 아니라, 입력 문맥에 의존하는 '공간적 가변 함수'라는 점입니다.

1. Instance Normalization을 통해 전역적인 특징을 제거하여 깨끗한 뼈대를 만듭니다.
2. 그 후, 문맥 맵(예: 세만틱 세그멘테이션, 위성 이미지 등)을 입력받는 작은 신경망(SPADE Block)이 픽셀 위치($s$)마다 최적의 정규화 모수($\gamma_s, \beta_s$)를 동적으로 산출합니다.
3. 최종적으로 이 모수들을 이용하여 **각 픽셀별로 서로 다른** 아핀 변환을 수행합니다.
